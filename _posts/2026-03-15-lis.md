LIS(Longest Increasing Subsequence, 가장 긴 증가하는 부분 수열)는 DP의 대표 문제다. N이 작으면 O(N²) DP로 충분하지만, N이 커지면 이분탐색을 활용한 O(N log N) 풀이가 필요하다. 두 풀이의 차이와 이분탐색 풀이가 왜 동작하는지를 정리한다.

---

## 문제 정의

> 수열 A가 주어졌을 때, 가장 긴 증가하는 부분 수열의 길이를 구하라.

부분 수열은 원래 수열에서 일부 원소를 **순서를 유지하면서** 선택한 것이다.

```
A = {10, 20, 10, 30, 20, 50}
LIS = {10, 20, 30, 50}  → 길이 4
```

---

## O(N²) DP 풀이

### dp 정의

```
dp[i] = arr[i]를 마지막 원소로 하는 LIS의 길이
```

### 점화식

```
dp[i] = max(dp[j] + 1)  for j < i if arr[j] < arr[i]
```

"i번째 원소 앞에 있는 원소 중, arr[i]보다 작은 것들의 dp값 + 1 중 최댓값"

### 구현

```python
import sys
input = sys.stdin.readline

n = int(input())
arr = list(map(int, input().split()))

dp = [1] * n  # 자기 자신만으로 이루어진 수열 길이 = 1

for i in range(1, n):
    for j in range(i):
        if arr[j] < arr[i]:
            dp[i] = max(dp[i], dp[j] + 1)

print(max(dp))
```

### 흐름

```
arr = {10, 20, 10, 30, 20, 50}

i=0: dp = [1, 1, 1, 1, 1, 1]
i=1: arr[0]=10 < arr[1]=20 → dp[1] = dp[0]+1 = 2
i=2: arr[0]=10 = arr[2]=10 → 조건 불만족
     dp = [1, 2, 1, 1, 1, 1]
i=3: arr[0]=10, arr[1]=20, arr[2]=10 모두 < 30
     dp[3] = max(dp[0], dp[1], dp[2]) + 1 = 3
i=4: arr[0]=10, arr[2]=10 < 20 → dp[4] = 2
     (arr[1]=20은 조건 불만족)
i=5: 앞의 모든 원소 < 50 → dp[5] = dp[3]+1 = 4

dp = [1, 2, 1, 3, 2, 4]
답: max(dp) = 4 ✅
```

### 한계

N ≤ 1,000,000일 때 O(N²) = 10¹² → **시간초과**

## O(N log N) 이분탐색 풀이

### 핵심 아이디어

`tails` 배열을 유지한다.

```
tails[i] = 길이 i+1인 증가 부분 수열의 마지막 원소 중 최솟값
```

각 원소마다 `tails`에서 **lower bound** (같거나 큰 값 중 가장 왼쪽)를 이분탐색으로 찾아 교체한다.

### 규칙

```
arr[x] > tails의 마지막 값  →  tails에 추가 (LIS 길이 +1)
arr[x] ≤ tails의 마지막 값  →  lower bound 위치의 값을 arr[x]로 교체
```

### 흐름

```
arr = {10, 20, 10, 30, 20, 50}

10 → tails: [10]
20 → 10보다 크므로 추가    tails: [10, 20]
10 → lower bound = 0, 교체 tails: [10, 20]  (변화 없음)
30 → 20보다 크므로 추가    tails: [10, 20, 30]
20 → lower bound = 1, 교체 tails: [10, 20, 30]  (변화 없음)
50 → 30보다 크므로 추가    tails: [10, 20, 30, 50]

답: len(tails) = 4 ✅
```

### 구현

```python
import sys
from bisect import bisect_left
input = sys.stdin.readline

n = int(input())
arr = list(map(int, input().split()))

tails = []
for x in arr:
    pos = bisect_left(tails, x)  # lower bound
    if pos == len(tails):
        tails.append(x)          # 맨 뒤보다 크면 추가
    else:
        tails[pos] = x           # 교체

print(len(tails))
```

`bisect_left`는 Python 표준 라이브러리로, lower bound를 C로 구현한 함수다. 직접 이분탐색을 짜는 것보다 빠르다.

### lower bound 직접 구현

```python
def lower_bound(tails, x):
    lo, hi = 0, len(tails)
    while lo < hi:
        mid = (lo + hi) // 2
        if tails[mid] < x:
            lo = mid + 1
        else:
            hi = mid
    return lo
```

---

## 왜 교체해도 길이가 맞냐?

`tails`는 실제 LIS가 **아니다**. 중간에 교체가 일어나면 LIS를 만족하지 않는 상태가 될 수 있다.

```
arr = {10, 20, 30, 15}

tails: [10] → [10, 20] → [10, 20, 30] → [10, 15, 30]

실제 LIS: {10, 20, 30}  길이 3
tails:    {10, 15, 30}  길이 3  ← 내용은 다르지만 길이는 같다
```

교체의 역할은 **이후 원소가 더 많이 들어올 가능성을 넓히는 것**이다. 더 작은 값으로 끝나는 수열을 유지하면, 다음에 오는 원소가 조건을 만족할 확률이 높아진다.

**길이만 구하는 문제에서는 내용물이 중요하지 않다.** 실제 LIS 수열을 출력해야 한다면 이 방법은 추가 처리가 필요하다.

---

## 두 풀이 비교

| | O(N²) DP | O(N log N) 이분탐색 |
|---|---|---|
| 시간복잡도 | O(N²) | O(N log N) |
| 공간복잡도 | O(N) | O(N) |
| N ≤ 5,000 | ✅ 통과 | ✅ 통과 |
| N ≤ 1,000,000 | ❌ 시간초과 | ✅ 통과 |
| 실제 LIS 출력 | ✅ parent 배열로 추적 가능 | ⚠️ 추가 처리 필요 |
| 구현 난이도 | 쉬움 | 이분탐색 이해 필요 |

---

## N 크기별 풀이 선택

```
N ≤ 5,000     →  O(N²) DP
N ≤ 1,000,000 →  O(N log N) 이분탐색
```

N을 보고 복잡도를 역산하는 것이 실전에서 중요하다. N = 1,000,000이 보이면 O(N²)는 즉시 포기하고 O(N log N) 이상을 생각한다.

---

## 정리

LIS는 단순한 DP 문제처럼 보이지만, N이 커지면 이분탐색과의 결합이 필요한 중급 이상의 문제다.

핵심은 두 가지다.

1. `tails[i]` = 길이 i+1인 증가 부분 수열의 **마지막 원소 최솟값** 유지
2. 교체는 길이에 영향을 주지 않고, 이후 원소가 더 많이 들어올 여지를 만든다

이 아이디어는 LIS 변형 문제(LDS, LNDS 등)에서도 동일하게 적용된다.
