## 들어가며

LIS(Longest Increasing Subsequence, **가장 긴 증가하는 부분 수열**)는 알고리즘에서 매우 유명한 문제다.  
특히 **동적 프로그래밍(DP)**을 공부할 때 거의 반드시 등장하는 대표적인 예제이기도 하다.

문제 자체는 단순하다.

> 수열이 주어졌을 때 **증가하는 부분 수열 중 가장 긴 것의 길이**를 구하라.

하지만 이 문제는 입력 크기에 따라 **완전히 다른 접근 방법**이 필요하다.

| 방법 | 시간복잡도 |
|---|---|
| DP | O(N²) |
| 이분탐색 + Greedy | O(N log N) |

N이 작다면 O(N²) DP로 충분하지만,  
N이 **수십만 ~ 백만** 수준으로 커지면 **이분탐색 기반 풀이**가 필요하다.

이 글에서는 다음 내용을 순서대로 정리한다.

1. **O(N²) DP 풀이**
2. **O(N log N) 이분탐색 풀이**
3. **왜 이분탐색 풀이가 동작하는지**

---

## 문제 정의

> 수열 A가 주어졌을 때 **가장 긴 증가하는 부분 수열(LIS)의 길이**를 구하라.

여기서 **부분 수열(subsequence)**은 원래 수열에서 일부 원소를 선택한 것이다.  
단, **원소의 순서는 유지해야 한다.**

예를 들어 다음 수열을 보자.

~~~
A = {10, 20, 10, 30, 20, 50}
~~~

가능한 증가 부분 수열 중 하나는 다음과 같다.

~~~
{10, 20, 30, 50}
~~~

따라서 LIS의 길이는

~~~
4
~~~

이다.

---

# O(N²) DP 풀이

## DP 정의

다음과 같이 정의한다.

~~~
dp[i] = arr[i]를 마지막 원소로 하는 LIS의 길이
~~~

즉 `dp[i]`는

> **i번째 원소에서 끝나는 증가 부분 수열의 최대 길이**

를 의미한다.

---

### 점화식

i번째 원소 앞에 있는 원소들을 살펴본다.

~~~
j < i
~~~

그리고 다음 조건을 만족해야 한다.

~~~
arr[j] < arr[i]
~~~

따라서 점화식은 다음과 같다.

~~~
dp[i] = max(dp[j] + 1)   (j < i, arr[j] < arr[i])
~~~

---

### 구현

~~~python
import sys
input = sys.stdin.readline

n = int(input())
arr = list(map(int, input().split()))

dp = [1] * n

for i in range(1, n):
    for j in range(i):
        if arr[j] < arr[i]:
            dp[i] = max(dp[i], dp[j] + 1)

print(max(dp))
~~~

---

### 동작 예시

~~~
arr = {10, 20, 10, 30, 20, 50}
~~~

DP 진행 과정

~~~
i=0 → dp = [1, 1, 1, 1, 1, 1]

i=1
10 < 20
dp[1] = 2

i=2
10 = 10 → 증가 아님

dp = [1, 2, 1, 1, 1, 1]

i=3
10 < 30
20 < 30
10 < 30

dp[3] = 3

i=4
10 < 20
10 < 20

dp[4] = 2

i=5
모든 원소 < 50

dp[5] = 4
~~~

최종 결과

~~~
dp = [1, 2, 1, 3, 2, 4]
~~~

따라서

~~~
max(dp) = 4
~~~

---

### 한계

이 방법의 시간복잡도는

~~~
O(N²)
~~~

예를 들어

~~~
N = 1,000,000
~~~

이라면 연산 수는

~~~
10¹²
~~~

수준이 된다.

즉 **실전 문제에서는 시간 초과가 발생한다.**

---

## O(N log N) 이분탐색 풀이

### 핵심 아이디어

이 풀이에서는 `tails` 배열을 유지한다.

~~~
tails[i] =
길이 i+1인 증가 부분 수열의
마지막 원소 중 최솟값
~~~

예를 들어

~~~
tails = [10, 20, 30]
~~~

이라면 의미는 다음과 같다.

~~~
길이 1 수열의 최소 끝값 → 10
길이 2 수열의 최소 끝값 → 20
길이 3 수열의 최소 끝값 → 30
~~~

이 배열을 유지하면 **LIS 길이를 효율적으로 관리할 수 있다.**

---

### 규칙

새로운 원소 `x`를 처리할 때 다음 규칙을 따른다.

#### 1️⃣ x가 tails 마지막 값보다 큰 경우

~~~
x > tails[-1]
~~~

~~~
tails.append(x)
~~~

---

#### 2️⃣ 그렇지 않은 경우

`tails`에서 **lower bound**를 찾는다.

~~~
tails[pos] ≥ x
~~~

그리고 교체한다.

~~~
tails[pos] = x
~~~

---

### 동작 예시

~~~
arr = {10, 20, 10, 30, 20, 50}
~~~

진행 과정

~~~
10 → [10]

20 → [10, 20]

10 → lower bound = 0
      [10, 20]

30 → [10, 20, 30]

20 → lower bound = 1
      [10, 20, 30]

50 → [10, 20, 30, 50]
~~~

따라서

~~~
len(tails) = 4
~~~

---

### 구현

~~~python
import sys
from bisect import bisect_left
input = sys.stdin.readline

n = int(input())
arr = list(map(int, input().split()))

tails = []

for x in arr:
    pos = bisect_left(tails, x)

    if pos == len(tails):
        tails.append(x)
    else:
        tails[pos] = x

print(len(tails))
~~~

---

## lower bound 직접 구현

~~~python
def lower_bound(arr, x):
    lo, hi = 0, len(arr)

    while lo < hi:
        mid = (lo + hi) // 2

        if arr[mid] < x:
            lo = mid + 1
        else:
            hi = mid

    return lo
~~~

---

## 왜 교체해도 길이가 맞을까?

핵심은 이것이다.

> `tails`는 실제 LIS가 아니다.

예를 들어

~~~
arr = {10, 20, 30, 15}
~~~

진행 과정

~~~
[10]
[10, 20]
[10, 20, 30]
[10, 15, 30]
~~~

결과

~~~
tails = [10, 15, 30]
~~~

하지만 실제 LIS는

~~~
{10, 20, 30}
~~~

이다.

내용은 다르지만 **길이는 동일하다.**

---

### 교체의 의미

교체의 목적은 하나다.

> **더 작은 값으로 수열을 끝내기**

예를 들어

~~~
[10, 20, 30]
~~~

보다

~~~
[10, 15, 30]
~~~

이 더 유리하다.

왜냐하면 이후에 들어올 숫자가

~~~
16, 17, 18 ...
~~~

같은 값이라면 **연결 가능성이 더 커지기 때문**이다.

---

## 두 풀이 비교

| | O(N²) DP | O(N log N) |
|---|---|---|
| 시간복잡도 | O(N²) | O(N log N) |
| 공간복잡도 | O(N) | O(N) |
| 구현 난이도 | 쉬움 | 중간 |
| 실제 LIS 출력 | 가능 | 추가 처리 필요 |

---

### N 크기별 풀이 선택

실전에서는 **N을 보고 알고리즘을 선택**해야 한다.

~~~
N ≤ 5,000      → O(N²) 가능
N ≤ 1,000,000  → O(N log N) 필요
~~~

---

## 정리

LIS 문제의 핵심은 다음 두 가지다.

1. `tails[i]`  
   → 길이 `i+1` 수열의 **최소 끝값 유지**

2. lower bound 교체  
   → **미래 확장 가능성을 유지**

이 아이디어는 다음 문제에도 그대로 적용된다.

~~~
LDS (Longest Decreasing Subsequence)
LNDS (Longest Non-Decreasing Subsequence)
~~~

LIS를 제대로 이해하면 **이분탐색 기반 DP 최적화 패턴**을 익힐 수 있다.
