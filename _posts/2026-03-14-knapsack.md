## 들어가며 
알고리즘을 공부하다 보면 **배낭 문제(Knapsack Problem)** 를 피하기 어렵다.  
DP의 교과서적인 예제이자, 실전에서도 다양한 형태로 변형되어 자주 등장한다.

이 글에서는 다음 세 가지를 한 번에 정리한다.

- **0/1 배낭 문제 (DP)**
- **분할 배낭 문제 (Greedy)**
- **배낭 문제의 변형 (BOJ 7579)**

핵심은 **문제의 구조를 보고 어떤 접근을 선택해야 하는지 판단하는 것**이다.

---

## 배낭 문제란?

> 무게 제한이 W인 배낭에 N개의 물건을 담을 때 가치의 합을 최대화하라.

각 물건은 다음 두 값을 가진다.

- **weight (무게)**
- **value (가치)**

여기서 중요한 차이는 하나다.

> **물건을 쪼갤 수 있는가?**

이 조건에 따라 문제는 크게 두 가지로 나뉜다.

---

# 1. 0/1 배낭 문제 (Dynamic Programming)

각 물건을 **넣거나(1) 넣지 않거나(0)** 두 가지 선택만 가능하다.

---

## 왜 그리디가 실패할까?

직관적으로는 다음 전략을 떠올릴 수 있다.

> 가치 / 무게 비율이 높은 것부터 담자

하지만 0/1 배낭에서는 이 전략이 항상 최적을 보장하지 않는다.

~~~
W = 50

물건:    A     B     C
무게:   10    20    30
가치:   60    100   120
비율:   6     5     4
~~~

그리디 선택

~~~
A + B = 160
~~~

하지만 실제 최적해는

~~~
B + C = 220
~~~

이다.

즉 **조합에 따라 결과가 달라지기 때문에** 모든 경우를 고려해야 한다.

그래서 **Dynamic Programming**을 사용한다.

---

## 점화식

~~~
dp[i][w] = i번째 물건까지 고려했을 때
           무게 w 이하로 담을 수 있는 최대 가치
~~~

선택은 두 가지다.

~~~
dp[i][w] = max(
    dp[i-1][w],                          # 물건 i를 선택하지 않음
    dp[i-1][w - weight[i]] + value[i]    # 물건 i를 선택
)
~~~

---

## 구현 (2D DP)

~~~python
n, W = map(int, input().split())
items = [tuple(map(int, input().split())) for _ in range(n)]

dp = [[0] * (W + 1) for _ in range(n + 1)]

for i in range(1, n + 1):
    w, v = items[i - 1]
    for j in range(W + 1):
        dp[i][j] = dp[i-1][j]
        if j >= w:
            dp[i][j] = max(dp[i][j], dp[i-1][j-w] + v)

print(dp[n][W])
~~~

---

## 공간 최적화 (1D DP)

~~~python
dp = [0] * (W + 1)

for w, v in items:
    for j in range(W, w - 1, -1):
        dp[j] = max(dp[j], dp[j - w] + v)

print(dp[W])
~~~

여기서 중요한 포인트가 하나 있다.

> **역순으로 순회해야 한다.**

---

## 왜 역순인가?

정순으로 순회하면 이미 갱신된 값을 다시 참조하게 된다.

~~~
dp[j] = dp[j-w] + v
~~~

이때 `dp[j-w]`가 **같은 물건으로 갱신된 값**일 수 있다.

그러면 같은 물건을 여러 번 사용하는 **완전 배낭(Unbounded Knapsack)** 이 된다.

따라서 **각 물건을 한 번만 사용하려면 반드시 역순 순회**가 필요하다.

---

# 2. 분할 배낭 (Greedy)

분할 배낭에서는 물건을 **쪼갤 수 있다.**

이 경우에는 그리디 전략이 항상 최적을 보장한다.

---

## 핵심 아이디어

~~~
가치 / 무게 비율이 높은 물건부터 담는다
~~~

남은 공간이 부족하면 **일부만 담는다.**

---

## 구현

~~~python
n, W = map(int, input().split())
items = [tuple(map(int, input().split())) for _ in range(n)]

items.sort(key=lambda x: x[1] / x[0], reverse=True)

total = 0.0

for w, v in items:
    if W <= 0:
        break

    if W >= w:
        total += v
        W -= w
    else:
        total += v * (W / w)
        W = 0

print(total)
~~~

---

## 0/1 vs 분할 배낭

| 문제 | 특징 | 해결 방법 |
|---|---|---|
| 0/1 배낭 | 물건을 쪼갤 수 없음 | DP |
| 분할 배낭 | 물건을 쪼갤 수 있음 | Greedy |

---

# 3. 배낭 문제 변형 (BOJ 7579)

실전 문제에서는 배낭 구조가 **조금 변형되어 등장**한다.

문제:

> N개의 앱이 실행 중이다.  
> 각 앱은 메모리 mᵢ를 사용하고 비활성화 비용 cᵢ가 있다.  
>  
> **M 이상의 메모리를 확보하면서 비용을 최소화하라.**

---

## 무엇이 다른가?

일반 배낭

~~~
무게 제한 안에서 가치 최대
~~~

이 문제

~~~
가치 하한을 만족하면서 비용 최소
~~~

즉 **제한과 목표가 뒤바뀐 구조**다.

---

## DP 재설계

~~~
dp[j] = 비용 j로 확보할 수 있는 최대 메모리
~~~

여기서 인덱스를 **비용**으로 설정한다.

이유

~~~
메모리 최대: 10,000,000
비용 최대: 10,000
~~~

메모리를 인덱스로 쓰면 배열이 너무 커진다.

---

## 구현

~~~python
import sys
input = sys.stdin.readline

n, M = map(int, input().split())
m = list(map(int, input().split()))
c = list(map(int, input().split()))

max_cost = sum(c)
dp = [0] * (max_cost + 1)

for i in range(n):
    for j in range(max_cost, c[i] - 1, -1):
        dp[j] = max(dp[j], dp[j - c[i]] + m[i])

print(min(j for j in range(max_cost + 1) if dp[j] >= M))
~~~

핵심은 동일하다.

> **0/1 배낭 → 역순 순회**

---

# 복잡도 분석

| 문제 | 시간복잡도 | 공간복잡도 |
|---|---|---|
| 0/1 배낭 (2D) | O(NW) | O(NW) |
| 0/1 배낭 (1D) | O(NW) | O(W) |
| 분할 배낭 | O(N log N) | O(N) |
| 앱 문제 | O(N × ΣC) | O(ΣC) |

---

# 정리

배낭 문제는 다음 흐름으로 접근하면 된다.

~~~
물건을 쪼갤 수 있는가?

Yes → Fractional Knapsack → Greedy
No  → 0/1 Knapsack → Dynamic Programming
~~~

DP를 설계할 때는 다음 세 가지를 확인한다.

1. 제한 조건 → 인덱스
2. 최적화 대상 → dp 값
3. 역순 순회 → 중복 선택 방지

배낭 문제는 다양한 변형이 있지만 **핵심 구조는 대부분 동일하다.**  
문제에서 무엇이 **제한 조건인지**, 무엇을 **최적화해야 하는지**만 파악하면 DP 상태를 설계할 수 있다.
