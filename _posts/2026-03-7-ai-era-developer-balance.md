## AI가 창의성마저 넘본다

"AI는 창의적이지 못하다"는 말, 이제 통하지 않는다.

Midjourney는 이 세상에 없던 그림을 그린다. GPT-4는 요구한 것 이상의 코드를 제안한다. Claude는 내가 생각 못한 엣지 케이스를 지적한다.

"모방은 창조의 어머니"라는 말이 있다. AI는 인류가 만든 모든 데이터를 학습했다. 빅데이터에서 패턴을 찾고, 새롭게 조합한다. 어쩌면 인간보다 더 창의적일 수도 있다.

---

## 실제로 AI가 놀라웠던 순간들

### 예상 이상의 제안

Three.js 최적화 물어봤을 때:
```
나: "렌더링 느린데 어떻게 하지?"

Claude: "Instanced Mesh 쓰세요. 
그런데 당신 코드 보니까 매 프레임마다 geometry 업데이트하시는데, 
static mesh는 미리 생성하고 dynamic한 것만 업데이트하면 더 빠를 겁니다."
```

내가 묻지도 않은 부분까지 봤다.

### 버그 패턴 발견

"왜 모바일에서만 터치 지연되지?"

GPT-4: "브라우저 더블탭 줌 때문일 겁니다. preventDefault() + passive: false 해보세요."

정확했다. 구글링 30분이 30초로 단축됐다.

### 디자인 인사이트

"게임 UI 개선안 줘"

Cursor: "'한 번 더' 버튼을 점수 바로 아래 두면 사용자 시선 흐름상 자연스럽습니다."

실제로 재시작률 45% → 78%.

## 할루시네이션은 줄어들고 있다

**2023년:** 존재하지 않는 함수 제안
**2024년:** "확신 못하면 말해줘" → "검색해볼게요"
**2025년:** o1, Claude 3.5 - 체인 오브 생각으로 검증

점점 정확해진다. 믿을 만해진다.

## 활용 능력이 핵심이다

AI가 못하는 것:
- ~~창의성~~ → AI도 창의적이다
- ~~코딩~~ → AI가 더 잘 짤 수도
- ~~디자인~~ → AI가 제안도 한다

**남은 것: AI를 활용하는 능력**

차이는 여기다:
- AI 혼자: 방향 없이 표류
- 인간 + AI: 목표 향해 전진

---

## 활용 능력이란?

### 1. 문제 정의

**나쁜 질문:**
"재미있는 게임 만들어줘"

**좋은 질문:**
"구글 두들 달 게임 클론. 그래프 기반 보드에 카드 배치. Phase Pair (1점), Full Moon Pair (2점), Lunar Cycle (N점) 스코어링. Chain Steal 메커니즘 추가해줘."

AI는 질문만큼 똑똑하다.

### 2. 반복과 개선

체인 애니메이션 예시:
```
1차: "체인 애니메이션 만들어줘" → 너무 빠름
2차: "400ms 간격으로 천천히" → 좋음, 하지만 밋밋
3차: "카드 확대 + 금색 테두리 + 사운드" → 완벽
4차: "놓은 카드부터 시작하게" → 직관적!
```

한 번에 완벽한 건 안 나온다. 대화하듯 개선하는 능력.

### 3. 도메인 지식

AdvChameleon 논문 쓸 때 AI가 제안한 코드:
```python
formants = librosa.piptrack(y, sr)  # 틀렸다
```

piptrack은 pitch tracking. formant는 따로 추출해야 한다.

**도메인 지식이 있어야 판단 가능하다.**

## AI 의존의 부작용

### 경험담: AI 없이는 아무것도 못하는 순간

어느 날 인터넷이 끊겼다. Cursor도, ChatGPT도 안 된다.

간단한 정렬 알고리즘 짜려는데... "어, quicksort가 어떻게 작동하더라?"

**30분 걸렸다.** AI 있으면 30초.

그 순간 깨달았다: 나는 알고리즘을 이해한 게 아니라, AI에게 물어보는 법만 배운 거였다.

---

## 의존의 위험성

### 1. 기초 실력이 안 쌓인다
```
Junior: "배열 정렬 어떻게 해요?"
AI: [코드 제공]
Junior: 복붙 → 작동 → 다음 문제

1년 후... 여전히 정렬 알고리즘 모름.
```

문제가 생기면 해결 못한다. 최적화 방향도 모른다.

### 2. 블랙박스 신드롬
```typescript
// AI 제안
const optimized = useMemo(
  () => data.map(x => transform(x)),
  [data, transform]
);
```

"useMemo가 뭐죠?" "모르겠는데 AI가 해줬어요"

실제로 이 코드는 `transform`이 함수라서 dependency에 넣으면 매번 재계산. **오히려 느려진다.**

### 3. 스스로 생각하지 않게 된다

**전:**
```
문제 → 고민 → 가설 → 실험 → 해결
```

**AI 의존 후:**
```
문제 → AI에게 물어봄 → 복붙 → 끝
```

문제 분석 능력, 가설 수립 능력, 깊이 있는 이해를 잃는다.

### 4. 창의성이 위축된다

항상 AI 답변부터 보면:
- 내 아이디어가 "AI보다 못해 보임"
- 시도 안 함
- 자기 생각을 못 믿게 됨

체인 애니메이션 고민 중 내 아이디어는 "카드 확대", AI는 "파티클 + 확대 + 회전" 제안. AI 답변에 압도당해 내 아이디어를 버렸는데, 실제로 해보니 단순한 게 더 나았다.

## 균형 잡기

### 원칙 1: 이해한 것만 쓴다
```
AI 코드 받음 → 이해 안 되는 부분? → "이게 왜 필요해?"
→ 이해될 때까지 → 그제서야 사용
```

### 원칙 2: 기초는 손으로

**AI 쓰지 말아야 할 때:**

1. 알고리즘 공부 - 직접 짜보기
2. 새로운 개념 학습 - 공식 문서 먼저
3. 디버깅 초반 - 30분 스스로 시도

고생해야 배운다.

### 원칙 3: 주 1회 AI 금지일

매주 하루는 AI 없이 코딩.

목적: 내 실력 확인, 의존도 체크, 기초 다지기.

### 원칙 4: 핵심은 직접

**AI에게:**
- 보일러플레이트
- 반복 코드
- 테스트 코드

**내가:**
- 핵심 알고리즘
- 비즈니스 로직
- 아키텍처 설계

## 건강한 AI 활용법

### 단계별 접근

**1단계: 스스로 시도 (30분)**
문제 발생 → 고민 → 시도

**2단계: AI에게 힌트 (10분)**
"방향만 알려줘, 코드는 말고"

**3단계: 구현은 반반**
AI가 초안 → 내가 수정 (복붙 금지, 타이핑하면서 이해)

**4단계: 검증과 개선**
AI 코드 → 테스트 → 문제 발견 → 내가 고침

### 위험 신호

이런 증상 있으면 위험:
- "AI 없으면 못 짜겠어"
- "어떻게 작동하는지 모르겠는데 잘 돼"
- "에러 나면 무조건 AI한테 물어봄"
- "알고리즘? 모르는데 AI가 해줘"

**하나라도 해당되면 AI 의존도 낮춰야 한다.**

---

## 실전 워크플로우

### 아침 (계획)
나: 오늘 할 일 정리 → AI: 우선순위 제안 → 나: 최종 결정

### 오전 (구현)
나: "이 기능 필요" → AI: 기본 코드 → 나: 핵심 로직 수정 → AI: 테스트 코드

### 오후 (디버깅)
나: "이 버그 뭐지?" → AI: 가능성 제시 → 나: 검증 및 수정

### 저녁 (정리)
나: 코드 리뷰 → AI: 개선 제안 → 나: 적용 여부 결정

**AI는 조수, 나는 감독.**

## 준비 전략

### 지금 당장

1. **AI 도구 마스터** - ChatGPT, Claude, Cursor 매일 쓰기
2. **기초 다지기** - 알고리즘, 시스템 디자인, 도메인 지식
3. **빠르게 만들기** - 아이디어 → 프로토타입 (1일 목표)

### 1-2년 내

1. **전문성 확보** - T자형 인재 (깊이 + 넓이)
2. **AI 활용 루틴** - 최적화된 협업 패턴 확립
3. **균형 유지** - 활용 능력 ↑ 동시에 기초 ↑

### 5년 후

1. **AI 전문가** - 최신 도구 능숙하게, 한계 정확히 알기
2. **문제 정의자** - 방향 설정 능력
3. **평생 학습자** - 배우는 속도가 경쟁력

## 나의 균형점

**AI 쓰는 것 (60%):** 보일러플레이트, 리팩토링, 테스트, 디버깅 힌트

**직접 하는 것 (40%):** 핵심 알고리즘, 아키텍처, 최적화 결정, UX 플로우

**결과:** 속도 2배, 실력 여전히 성장, 이해도 깊이 유지

**균형이 핵심이다.**

---

## 결론

AI는 양날의 검이다.

**잘 쓰면:** 생산성 ↑, 학습 속도 ↑, 창의성 증폭
**잘못 쓰면:** 기초 약화, 의존성 ↑, 사고력 퇴화

경쟁은 AI vs 인간이 아니다.
**AI 쓰는 인간 vs AI 안 쓰는 인간**

활용하되 의존하지 말고, 빠르게 가되 이해하며 가고, AI를 쓰되 스스로 생각하자.

그게 AI 시대 생존법이다.
